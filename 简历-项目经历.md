# 简历 — 项目经历

## 老年陪伴AI — 音频感知子系统

**角色**：算法工程师（独立开发）&emsp; **周期**：2026.06 — 至今

### 项目概述

为老年陪伴机器人构建**多模态感知系统**。独立负责听觉子系统，实现从麦克风采集 → 声音事件检测（SELD）→ 情绪识别 → 多模态融合的端到端管道，并与视觉子系统完成交叉验证集成。系统在 GPU 上达到 **200ms 端到端延迟**，融合后跌倒检测置信度从单模态 0.4 提升至 **0.9+**，满足居家安全监护的 500ms 响应要求。

### 技术栈

`Python` `PyTorch` `fairseq` `FastAPI` `PyAudio` `scipy` `wav2vec 2.0` `多模态融合` `声源定位(DOA)`

### 工作内容与成果

**1. 设计并实现音频采集与预处理管道（AudioPipeline）**

- 基于 PyAudio 构建跨平台实时音频采集模块，支持单/双/四通道麦克风自适应识别
- 实现采样率转换（抗混叠重采样）、自适应增益控制（滑窗 RMS，限幅 0.1~10x）、伪 Ambisonics 空间合成（单声道 → HRTF 四通道 B 格式）
- 处理流水线优化至 **5ms 预处理延迟**，支持 100ms 帧长的实时输出

**2. 搭建 w2v-SELD 声音事件检测推理引擎**

- 实现 fairseq Wav2Vec2Model 的加载与推理封装，设计**双策略 fallback 机制**（fairseq API → torch.load state_dict），兼容不同 checkpoint 格式
- 实现 SED 后处理：sigmoid 概率转换 → 阈值激活 → 连续帧合并 → 最小事件时长过滤，支持按类别独立调参
- 建立配置管理三层体系（dataclass 默认值 → YAML 文件 → 环境变量覆盖），支持零代码调参
- 封装 FastAPI REST 服务（`POST /v1/audio/events` + 健康检查），支持文件上传与字节流双模式输入

**3. 设计多模态融合层（Audio × Vision Cross-Validation）**

- 定义 **AudioPerception / VisualPerception 标准化感知输出契约**，通过时间戳对齐实现异构传感器数据同步
- 构建 **4 象限融合矩阵**（听/视觉异常 × 双信号），实现交叉验证：双信号吻合时置信度叠加（fall 从 0.4→0.9+），单信号矛盾时自动降级
- 融合器集成 **情绪加成**（听觉情绪 + 面部表情双源 distress 评分）与 **DOA-BBox 空间一致性校验**（声源方位 ↔ 人物位置映射）
- 实现 **AdaptiveFusion 自适应融合器**：根据昼夜时段动态调整报警阈值（夜间敏感度 ×1.5），结合 Persona 行为基线降低活跃时段误报
- 输出标准化 FusionResult（紧急等级 + 各维度置信度 + 交叉验证标记），供下游决策和记忆系统消费

**4. 训练流程设计与数据工程**

- 设计从数据准备到微调评估的完整训练管道：标注音频 → DCASE SELD 标准格式转换 → 伪空间数据增强 → seld_wav2vec2 微调
- 数据方案：公开数据集打底 + 自录场景适配 + **RIR 房间脉冲响应卷积合成**做数据增强，解决真实四通道标注数据稀缺问题
- 训练策略：分层学习率（backbone 1e-6 / 任务头 1e-4）防止灾难性遗忘，按文件切分 70/15/15 防止标签泄漏

**5. 云端 GPU 部署与工程化**

- 将项目模块化拆分为**独立 Python 包**（src-layout，pip install -e），实现采集、推理、融合三层解耦部署
- 编写 ModelScope 云端 GPU 一键训练脚本，打通 git clone → 环境安装 → 权重下载 → 微调训练 → 模型导出的全流程
- 建立 **91 个单元测试 + Mock 模型测试框架**，实现在无 GPU/无权重条件下对全流水线的 CI 验证

### 难点攻克

- **多模态融合矛盾**：听觉和视觉信号冲突时如何决策 → 4 象限交叉验证矩阵 + 空间一致性校验 + 历史平滑，单信号矛盾自动降级
- **单麦克风空间定位**：基于 HRTF 原理的伪 Ambisonics 合成（高通→前方、延迟→左右、低通→上下），声源方向与视觉 bbox 做粗粒度一致性检查
- **依赖兼容**：fairseq 与 PyTorch 版本强耦合，设计双策略模型加载 + torchaudio/soundfile 双后端音频解码
- **实时性**：边缘设备 CPU 推理延迟从 500ms → 目标 200ms，方案为 INT8 量化 + 流式 Transformer 逐帧推理
- **数据稀缺**：采用 RIR 房间脉冲响应卷积合成四通道训练数据，配合弱监督标注降低人工成本
