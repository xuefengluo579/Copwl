## 1. 模块概述

基于 **wav2vec 2.0** 预训练模型的声音事件检测与定位 (SELD) 推理引擎。输入四通道 Ambisonics B 格式音频（16kHz），输出结构化事件列表（类型、置信度、时间戳、声源方位）。

**一句话定位**：给上层决策模块（如老年陪伴AI的思考模块）提供「听到了什么、在哪、动静多大」的结构化感知数据。

### 核心能力

| 维度 | 说明 |
|------|------|
| 事件类型 | speech / fall / glass_break / knock / object_drop / footstep / door_slam（7 类） |
| 输入格式 | 4 通道 Ambisonics B，16kHz，float32 |
| 输出格式 | `List[DetectedEvent]`，含事件类型、置信度、起止时间、DOA 方位 |
| 推理模式 | 批量推理 + 流式推理（滑动窗口） |
| 部署方式 | Python 包 `import seld` + REST API (`POST /v1/audio/events`) |
| 硬件要求 | GPU ≥ 4GB (RTX 3060+) 或 CPU 回退 |

### 当前阶段：推理就绪，微调待做

本交付仅包含推理流水线，不含模型微调代码。使用预训练权重即可检出 7 类 DCASE 标准事件。

## 3. 快速接入（3 分钟）

### 3.1 安装

```bash
cd 音频识别/
pip install -e .
```

### 3.2 下载权重

```bash
python scripts/download_weights.py --variant base
```

### 3.3 最小示例

```python
from seld import SELDConfig, W2vSELDEngine
import torchaudio
import numpy as np

# 1. 加载配置
config = SELDConfig.from_yaml("config/default.yaml")

# 2. 初始化引擎
engine = W2vSELDEngine(config)

# 3. 加载音频（自动处理重采样和单通道→Ambisonics转换）
audio, sr = torchaudio.load("sample.wav")
if sr != 16000:
    audio = torchaudio.transforms.Resample(sr, 16000)(audio)
if audio.shape[0] == 1:
    from seld import mono_to_ambisonics
    audio = mono_to_ambisonics(audio)

# 4. 推理
events, elapsed_ms = engine.infer(audio.numpy())
for e in events:
    print(f"{e.event.value}: conf={e.confidence:.2f}, "
          f"t=[{e.t_start_ms}..{e.t_end_ms}]ms")
```

## 4. 公开 API 清单

### 4.1 核心类

| 类 | 说明 | 主要方法 |
|----|------|----------|
| `W2vSELDEngine(config)` | 推理引擎。初始化时加载模型。 | `infer(audio) → (events, ms)`<br>`preprocess(audio) → tensor`<br>`postprocess(logits) → events` |
| `StreamingSELDEngine(config)` | 流式引擎。维护滑动窗口缓冲区。 | `feed(chunk) → events or None`<br>`reset()` |
| `SELDConfig` | 总配置类。 | `from_yaml(path) → SELDConfig`<br>`from_env() → SELDConfig`<br>`validate()` |

### 4.2 配置子类

| 类 | 关键字段 |
|----|----------|
| `ModelConfig` | `model_path`, `checkpoint_path`, `device` (auto/cuda/cpu), `use_fairseq` |
| `AudioConfig` | `sample_rate` (16000), `num_channels` (4), `frame_duration_ms` (100) |
| `InferenceConfig` | `event_threshold` (0.5), `min_event_duration_ms` (100), `batch_size` (1) |
| `StreamingConfig` | `window_seconds` (2.0), `hop_seconds` (0.5) |

### 4.3 数据模型

| 类 | 字段 |
|----|------|
| `DetectedEvent` | `event: EventType`, `confidence: float`, `t_start_ms: int`, `t_end_ms: int`, `doa: Optional[DOAVector]` |
| `DOAVector` | `x: float`, `y: float`, `z: float` + `.azimuth`/`.elevation` 属性 |
| `DetectionResult` | `events: List[DetectedEvent]`, `inference_time_ms: float`, `audio_duration_ms: int` |
| `HealthStatus` | `status`, `model_loaded`, `device`, `version`, `uptime_seconds` |

### 4.4 音频工具函数

## 6. REST API 参考

### GET /health

```json
{"status":"ok","model_loaded":true,"device":"cuda","version":"0.1.0","uptime_seconds":120.5,"model_name":"w2v_seld_base.pt"}
```

### POST /v1/audio/events

- **输入**: `multipart/form-data`，字段名 `file`，支持 WAV/FLAC/OGG
- **限制**: ≤ 30 秒，自动重采样到 16kHz，单通道自动转四通道
- **输出**: `DetectionResult`

```json
{
  "events": [
    {
      "event": "fall",
      "confidence": 0.94,
      "t_start_ms": 0,
      "t_end_ms": 450,
      "doa": {"x": 0.32, "y": -0.15, "z": 0.08}
    }
  ],
  "inference_time_ms": 52.3,
  "audio_duration_ms": 2000
}
```

## 8. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 包布局 | `src/` layout | 代码与配置隔离，pip install -e . 开发友好 |
| 配置方案 | dataclass + YAML + 环境变量三层覆盖 | 配置项仅 ~10 个，不需要重型框架 |
| 模型加载 | fairseq API → state_dict 双策略 fallback | 兼容不同 checkpoint 格式 |
| 设备选择 | `"auto"` → CUDA > MPS > CPU | 零配置智能降级 |
| 流式缓冲 | `np.roll` 循环数组 | 避免 deque 在热路径的 Python 开销 |
| 音频解码 | torchaudio → soundfile 双解码器 fallback | torchaudio 2.9+ 需要 torchcodec |
| DOA 输出 | 可选字段，`clamp(-1,1)` 安全裁剪 | Mock 模型生成随机值时不出错 |
| 最小事件时长 | 默认 100ms，可配置 | 过滤单帧瞬态噪声，降低误报 |

## 9. 已知限制 & 下一步

### 当前限制

| 限制 | 影响 | 解决方案 |
|------|------|----------|
| 仅支持 7 类 DCASE 事件 | 自定义事件需微调 | 见下一步「模型微调」 |
| DOA 分支未完整实现 | 方位信息可能不准 | 需根据 seld_wav2vec2 实际 forward 签名适配 |
| 无 ONNX/TensorRT 导出 | CPU 推理较慢（~500ms） | `torch.onnx.export` + ONNX Runtime |
| 无流式 API 端点 | API 仅支持文件上传 | 增加 WebSocket 端点 |
| 模型加载失败时服务降级 | API 返回 503 | 当前策略是显式报错，不静默吞 |

### 下一步任务建议

1. **下载权重验证** — 运行 `download_weights.py` → 跑 `pytest -m slow`
2. **编写微调脚本** — 基于 seld_wav2vec2 仓库的 finetune.py，用自有数据集微调
3. **DOA 集成** — 阅读 seld_wav2vec2 代码，确认 model.forward() 的实际输出签名
4. **性能基准测试** — 在不同硬件（RTX 3060 / Jetson / CPU）上测试延迟
5. **WebSocket 流式 API** — 增加 `/v1/audio/stream` 端点，支持实时音频推送
